# Game Blueprint

Use this reference when drafting a full game package.

## Output Package

For a complete game, produce:

1. Game overview
   - Title
   - Occasion and audience
   - Player count
   - Duration
   - Tone
   - Learning or experience goal
   - Core question players must answer

2. Role map
   - Roles and relationships
   - Communication restrictions, if any
   - Role-specific pressure points

3. Host guide
   - Opening speech
   - True timeline
   - Responsibility ranking
   - Rules and phase timing
   - Evidence release plan
   - Steering principles
   - Final reveal script
   - Debrief questions

4. Character scripts
   - One private script per player
   - Immersive opening story
   - Public stance
   - Hidden issue
   - Known facts
   - Goals
   - Suggested phrases
   - Main culprit evasions, if applicable

5. Evidence cards
   - Public cards
   - Investigation cards
   - Late decisive cards
   - Optional bonus cards for longer games

## Responsibility Model

Use this model to keep the mystery fair.

| Layer | Meaning | Example |
|---|---|---|
| Primary cause | The direct mechanism that created the bad outcome | A pricing function returns the wrong amount |
| Secondary miss | A check that should have caught it | Testing misses the risky scenario |
| Ambiguity | A decision or spec that left room for error | The requirement omits boundary rules |
| Pressure | Incentive or deadline that made mistakes more likely | Leadership pushes a fixed launch date |
| Amplifier | Makes damage larger but did not create the defect | A link is shared before full monitoring |
| Red herring | Suspicious, but not causally central | A design change looks related but only affects display |

The final answer should identify the primary cause, while acknowledging the other layers.

## Suggested Phase Structure

Default 45-minute version:

1. Character reading: 5 minutes
2. Host opening + public evidence: 3 minutes
3. Private communication: 8 minutes
4. First public statements: 9 minutes
5. Questioning: 6 minutes
6. Investigation choice + evidence release: 4 minutes
7. Second private communication: 4 minutes
8. Final statements: 4 minutes
9. Vote + reveal: 5 minutes

Adjust for player count:

- 4-5 players: one private communication round, 8-10 evidence cards.
- 6-7 players: two communication rounds, 12-15 evidence cards.
- 8+ players: team roles or paired departments, 15-20 evidence cards.

## Role Design Checklist

Each role needs:

- A reason to speak.
- A reason to hide something.
- A reason to suspect at least one other role.
- At least one evidence card that pressures them.
- At least one plausible defense.
- Partial information that helps solve the case if shared.

Avoid:

- One role being purely innocent with nothing to protect.
- One clue proving the culprit too early.
- A culprit who can only deny and has no plausible alternative theory.
- A solution that depends on obscure domain knowledge.

## Final Quality Checks

Before delivering:

- Can a host explain the true story in 90 seconds?
- Can each player understand their script in 5 minutes?
- Does each evidence card look like an artifact?
- Are host-only notes absent from player-facing cards?
- Is the culprit pressured by multiple clues, not one blunt accusation?
- Are non-culprits pressured without being unfairly doomed?
- Does the final reveal distinguish creation, detection failure, and amplification?
